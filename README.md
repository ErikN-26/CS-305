# CS-305
For my CS 305 portfolio artifact, I chose to include my completed Artemis Financial secure software work. Artemis Financial was a financial consulting company that needed help improving the security of its web application. The company wanted to protect sensitive customer and financial data, so the main issue I had to address was identifying security vulnerabilities and applying secure coding practices to reduce risk.

One thing I did well during the vulnerability assessment was carefully reviewing the application for security weaknesses, including dependency vulnerabilities, insecure communication, and areas where better validation or protection was needed. Secure coding is important because software that handles financial or personal data can become a major risk if it is not protected correctly. Strong software security adds value to a company by protecting customer trust, reducing the chance of data breaches, and helping the business avoid legal, financial, and reputation damage.

The most challenging part of the vulnerability assessment was working with the dependency-check report and understanding which vulnerabilities were real issues and which ones could be false positives. Even though this part was challenging, it was also helpful because it showed me how important it is to use security tools carefully instead of just assuming every warning has the same level of risk.

I increased layers of security by using secure communication with HTTPS/TLS, reviewing dependencies, applying secure coding practices, and making sure the application used stronger security controls. In the future, I would use tools like OWASP Dependency-Check, vulnerability databases such as the NVD, manual code review, and secure coding standards to assess vulnerabilities and decide which mitigation techniques are best.

To make sure the code and software application were functional and secure, I tested the application after making changes and reviewed the results from the dependency check. After refactoring the code, I checked that the application still ran correctly and that the security changes did not introduce new errors or vulnerabilities. This helped confirm that the application was improved without breaking its main functionality.

Some resources, tools, and coding practices I used that will help me in future assignments include OWASP Dependency-Check, Maven, Java, Spring Boot, secure coding principles, vulnerability reports, and manual code review. I also learned the importance of documenting security decisions clearly so that another developer or employer can understand what was found and how it was addressed.

From this assignment, I could show future employers my vulnerability assessment report, secure software practices report, dependency-check results, and refactored secure code. These artifacts show that I can identify security risks, explain why they matter, apply mitigation strategies, and document my work in a professional way. This project is a good example of my growing skills in secure software development.
